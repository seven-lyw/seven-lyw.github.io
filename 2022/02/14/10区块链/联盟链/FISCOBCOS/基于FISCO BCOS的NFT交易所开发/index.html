基于FISCO BCOS的NFT交易所开发

简介

NFT简介

NFT（非同质代币）是一种加密代币。企业家和开发人员使用以太坊等公共区块链平台创建它们。NFT 与其他加密代币之间存在一些关键差异，如下所示：

  • NFT 是一种独特的加密代币。一个比特币与另一个比特币 (BTC) 完全相同，但是，每个 NFT 都是独一无二的。
  • 你不能将 NFT 分成更小的部分。另一方面，您可以将比特币或以太坊 (ETH) 分成更小的部分。

NFT 和其他类型的加密令牌之间存在一些相似之处。它们如下：

  • 开发人员在铸造加密代币时使用以太坊或 Nifty Gateway 等区块链生态系统。这适用于 NFT。
  • 您可以通过对某些计算机程序进行编码来创建加密令牌。NFT 也是如此。
  • 可以交易加密代币。NFT 也是如此。

基于区块链的游戏CryptoKitties中的数字小猫是 NFT 的突出例子。这款基于以太坊区块链的在线游戏允许玩家交易和收集数字小猫。其他流行的例子如下：

  • Axie Infinity 是一款 NFT 在线视频游戏，截至 2021 年夏季的销售额已超过 4200 万美元。
  • Cryptopunks 是以太坊区块链上首批不可替代的代币之一。
  • NBA 为官方授权的收藏品推出了 NBA Top Shot。
  • 著名数字艺术家 Beeple 推出了名为“Everydays – The first 5000 days”的 NFT。这是一幅艺术作品的拼贴画。佳士得最近以 6900 万美元的价格出售了他的艺术收藏品。
  • Axie Infinity 是一款 NFT 在线视频游戏，截至 2021 年夏季，其交易量已创纪录，销售额已超过 4200 万美元。
  • 无聊猿游艇俱乐部是 10000 个 NFT 的集合。

什么是NFT市场

NFT 市场是交易 NFT 的在线市场。NFT 托管在区块链网络平台上，允许出售从房地产和数字物品（如头像等）到借出数字资产和货币的所有东西。

  • 以太坊——最受欢迎的平台，也是估值第二高的加密货币。托管许多最流行的 Defi 解决方案、游戏和允许 NFT 销售等的市场。
  • Solana – 类似以太坊的平台，目前托管 400 多个项目，具有广泛的用例。

NFT 市场的运作方式如下：

  • 用户在基于区块链技术的平台上注册并选择一个数字钱包。
  • 数字收藏品的创建者创建他们的数字资产、创建 NFT 并上传数字资产。
  • 电子商务平台的产品列表功能显示数字收藏品。
  • 卖家可以指定固定价格或进行拍卖。
  • 电子商务市场帮助用户执行销售和拍卖。
  • 最后，在线商店处理数字资产和货币的转移。

流行的NFT市场有哪些？

OpenSea

OpenSea是著名的 NFT 市场。您可以在这里找到不同种类的数字收藏品。这些包括艺术品、交易卡、与体育相关的数字收藏品等。

您可以使用 OpenSea 创建一个帐户。你可以为你的数字收藏品 NFT 并出售它们。OpenSea 允许您搜索不同种类的 NFT，并且您可以购买它们。

Rarible

Rarible是另一个流行的 NFT 市场。它提供了 NFT 市场的所有标准功能。您可以创建您的帐户，随后，您可以创建数字收藏品。Rarible 允许您出售它们。您可以搜索收藏品并购买它们。

SuperRare

SuperRare是一个数字艺术在线市场平台。这个 NFT 市场的开发人员在以太坊上构建了它。SuperRare 提供 NFT 市场提供的标准功能。您可以创建您的帐户、创建数字艺术品并在 SuperRare 上出售它们。您还可以浏览数字艺术品并购买它们。

其他出售 NFT 和 NFT 收藏品的数字拍卖行的流行示例包括 Nifty Gateway 和 Decentraland。

需求

NFT平台功能

  • 电子商务店面；
  • 搜索功能，包括高级功能；
  • 搜索时的各种过滤器；
  • 产品列表的创建；
  • 显示产品列表的状态；
  • 购买功能，包括购物车、结账等功能；
  • 拍卖功能；
  • 与流行的数字钱包集成；
  • 支付功能，包括与 PayPal、Stripe 等支付网关的集成；
  • 线下活动同步；
  • 产品评级能力。

NFT平台商业模式

  • 你的目标受众是谁？
  • 你的关键价值主张是什么？
  • 您想运营多供应商市场吗？
  • 你想像优步那样在叫车领域做一个聚合器吗？
  • 您想使用哪些不同的定价机制？
  • 你会收取上市费吗？
  • 您打算利用共享经济的哪些特征？

您的团队应该使用适当的工具，例如：

  • 市场调查;
  • 采访不同的利益相关者。

该团队应确定项目范围。这应该包括以下内容：

  • 功能要求：您需要最终确定要提供的功能。有效地记录它们。按照我们的指南编写功能规范。
  • 非功能性需求：确定非功能性需求(NFR)，如可扩展性、性能等。有效地记录它们。
  • 平台：决定是否与电子商务网站一起构建移动应用程序。确定您要定位的平台，例如 Android、iOS 等。

MVP

您想验证您的商业理念，因此，您需要市场反馈。为此启动 MVP（最小可行产品）。MVP 是一种功能性产品。它的功能比最终产品少。但是，MVP 具有足够的功能来获得市场反馈。您可以根据反馈改进您的产品。

虽然您可能已经决定提供许多功能，但您将在 MVP 中提供较少的功能。您如何确定 MVP 中的这些功能？你需要优先考虑它们。为此，请使用以下工具：

  • 《痛苦与收获图》；
  • “优先矩阵”。
